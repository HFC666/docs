    # 强化学习介绍

    # 强化学习的特点

    是什么使得强化学习不同于其他机器学习算法？

    • 没有监督数据，只有奖励(reward)信号
    • 奖励信号不一定是实时的，可能存在延迟
    • 时间是一个重要因素
    • 智能体(Agent)当前的动作(Action)影响后续接收到的数据

    # 强化学习基本要素

    # 奖励

    • 奖励RtR_tR​t​​是一个反馈信号，是一个标量。
    • 反映智能体(Agent)在时间步ttt工作如何。
    • 智能体的工作就是最大化累积奖励
    • 强化学习主要基于奖励假设

    奖励假设：所有问题的目标都可以被描述成最大化期望的累积奖励。

    # 序列决策

    • 目标：选择一定的动作序列以最大化未来的总体奖励
    • 智能体的行为可能是一个很长的动作序列
    • 大多数奖励是延迟的
    • 宁愿牺牲即使(短期)奖励以获得更多的长期奖励

    # 智能体

    # 智能体与环境

    • 智能体在每个时间步ttt：

      • 接收观测OtO_tO​t​​
      • 接收标量奖励信号RtR_tR​t​​
      • 执行动作(Action)AtA_tA​t​​

    • 环境：

      • 接收动作AtA_tA​t​​
      • 产生观测Ot+1O_{t+1}O​t+1​​
      • 产生标量奖励信号Rt+1R_{t+1}R​t+1​​

    # 历史与状态

    历史是观测、行动和奖励的序列：

    Ht=O1,R1,A1,O2,R2,A2,⋯,Ot−1,Rt−1,At−1,Ot,RtH_t = O_1,R_1,A_1,O_2,R_2,A_2,\cdots,O_{t-1},R_{t-1},A_{t-1},O_t,R_t H​t​​=O​1​​,R​1​​,A​1​​,O​2​​,R​2​​,A​2​​,⋯,O​t−1​​,R​t−1​​,A​t−1​​,O​t​​,R​t​​

    根据这个历史可以决定接下来会发生什么：

    • 智能体选择行动
    • 环境选择观测及奖励

    状态是一种用于确定接下来会发生的事情(行动、观察、奖励)的信息，是关于历史的函数：

    St=f(Ht)S_t = f(H_t) S​t​​=f(H​t​​)

    # 环境状态
    • 环境状态stes_t^es​t​e​​是环境的内部状态
      • 用来确定下一个观测/奖励：如游戏机内的观测，我们的得分奖励
    • 环境状态通常对智能体是不可见的：比如游戏机怎么运行我们是不知道的
    • 即使SteS_t^eS​t​e​​可见，大都包含大量不相关的信息
    # 智能体状态
    • 智能体状态StaS_t^aS​t​a​​是智能体内部对信息的表达
      • 包括智能体可以使用的、决定未来动作的所有信息
      • 是强化学习算法使用的信息
    • 智能体状态是历史的函数：Sta=f(Ht)S_t^a = f(H_t)S​t​a​​=f(H​t​​)
    # 信息状态

    信息状态，也叫马尔可夫状态，包含历史上所有有用的信息。

    马尔可夫性：状态StS_tS​t​​具有马尔可夫性，当且仅当P[St+1∣St]=P[St+1∣S1,⋯,St]\mathbb{P}[S_{t+1}\mid S_t] = \mathbb{P}[S_{t+1}\mid S_1,\cdots,S_t]P[S​t+1​​∣S​t​​]=P[S​t+1​​∣S​1​​,⋯,S​t​​]

    给定当前的状态，将来与历史无关：

    H1:t→St→Ht+1:∞H_{1:t}\rightarrow S_t \rightarrow H_{t+1:\infty} H​1:t​​→S​t​​→H​t+1:∞​​

    • 智能体状态=其前面三个元素构成的序列：挨电
    • 智能体状态=每个元素发生频率：奶酪
    • 智能体状态=前面所有状态的序列：？
    # 完全可观测的环境

    完全可预测：智能体可以直接观察到全部环境状态：

    Ot=Sta=SteO_t = S_t^a = S_t^e O​t​​=S​t​a​​=S​t​e​​

    智能体状态 = 环境状态 = 信息状态

    正式地说，这是马尔可夫决策过程(MDP)

    # 部分可观测的环境

    智能体可以观测到环境的部分：

    • 打麻将
    • 斗地主

    智能体状态不等于环境状态

    真实地说，这是部分可观测马尔可夫决策过程(POMDP)

    # 智能体组成

    智能体由下述三个组件的一个或多个组成：

    • 策略(Policy)：智能体的行为函数。输入为状态，输出为行动决策。
    • 价值函数(Value function)：评估每个状态或行动有多好
    • 模型(Model)：智能体对环境的表示，是智能体眼里的环境。

    # 策略

    是学习智能体在特定时间的行为方式：

    • 是从状态到行动的映射
    • 确定性策略：a=π(s)a = \pi(s)a=π(s)
    • 随机策略：π(a∣s)=P(At=a∣St=s)\pi(a\mid s) = P(A_t=a\mid S_t=s)π(a∣s)=P(A​t​​=a∣S​t​​=s)

    # 价值函数

    价值函数是对未来累积奖励的预测：

    • 用于评估在给定的策略下状态的好坏
    • 可以用于选择动作

    Vπ(s)=Eπ[Rt+1+γRt+2+γ2Rt+3+⋯∣St=s]V_\pi(s) = \mathbb{E}_\pi[R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3}+\cdots \mid S_t=s] V​π​​(s)=E​π​​[R​t+1​​+γR​t+2​​+γ​2​​R​t+3​​+⋯∣S​t​​=s]

    # 模型

    模型用于模拟环境的行为，构建环境的动态特性

    解决下述两个问题：

    • 状态转移概率：用来预测环境的下一个状态：Pss′a=P[St+1=s′∣St=s,At=a]\mathcal{P}_{ss^\prime}^a = \mathbb{P}[S_{t+1} = s^\prime\mid S_t = s,A_t=a]P​ss​′​​​a​​=P[S​t+1​​=s​′​​∣S​t​​=s,A​t​​=a]
    • 奖励：预测环境给出的下一个即时奖励：Rsa=E[Rt+1∣St=s,At=a]\mathcal{R}_s^a=\mathbb{E}[R_{t+1}\mid S_t=s,A_t=a]R​s​a​​=E[R​t+1​​∣S​t​​=s,A​t​​=a]

    环境真实的运行机制不称为模型，称为环境动力学。

    # 智能体分类

    基于策略的更新与学习方法，强化学习方法可以分为：

    • 基于价值函数
    • 基于直接策略搜索(无价值函数)
    • 基于执行者-评论者

    根据强化学习算法是否依赖模型，强化学习方法可以分为：

    • 基于模型的强化学习算法
    • 无模型的强化学习算法

    根据环境返回的回报函数是否已知，强化学习方法可以分为：

    • 正向强化学习算法
    • 逆向强化学习算法：从专家的示例中学习回报函数

    # 强化学习问题

    # 学习与规划

    序列决策中的两个基本问题：

    • 强化学习
      • 环境初始未知
      • 智能体不断与环境交互
      • 智能体提升它的策略
    • 规划
      • 环境模型已知
      • 智能体根据Model进行计算，不进行额外交互
      • 智能体提升它的策略

    # 探索和利用

    • 强化学习类似于一个试错的学习
    • 智能体从其与环境的交互中发现一个好的策略
    • 在是错过程中不会损失太多奖励
    • 探索会发现有关环境的更多信息，有选择地放弃某些奖励
    • 利用已知信息来最大化回报，强调开发利用已有的信息
    • 探索和利用是决策需要平衡的两个方面

    # 预测与控制

    • 预测：评估未来
      • 策略已经给定
    • 控制：最大化未来
      • 找到最优的策略
