    # 绪论

    # 不同类型的任务

    • 节点级别
    • 子图级别
    • 边级别：
    • 图级别预测、图生成

    # 经典的图学习任务

    • 节点预测：预测节点的性质，如网络用户的分类
    • 联系预测：预测两个节点之间是否有缺失的边，如知识图的生成
    • 图分类：将不同的图进行分类，如我们用图来表示原子，对原子的性质进行预测
    • 聚类：检测节点是否来自子图：例如社交圈检测
    • 其他任务
      • 图生成：药物的发现
      • 图进化(Graph evolution)：物理模拟

    # 图表示的选择

    一个网络的组成

    • 对象: 节点, 顶点 NNN
    • 联系: 连接, 边 EEE
    • 系统: 网络, 图 G(N,E)G(N,E)G(N,E)

    图是一门通用语言。 我们可以用相同的图来表示不同的应用场景： 如上面三个情景都可以用右下角的图表示。

    我们需要选择一个合适的表示：

    • 如果你想将在一起工作的人连接起来，你将会建立一个职业网络
    • 如果你想要将相互引用的科学文章连接起来，你将会建立一个引用网络。 建立合适的图非常重要。

    我们怎么定义一个图？

    • 怎么构建一个图
      • 节点是什么
      • 边是什么？
    • 在给定领域或问题上建立合理的网络决定了你正确运用这些网络的能力。
      • 在一些情况下存在一个唯一的、明确的表示
      • 在其他情况下，表示可能不止有一种
      • 你制定连接的方式将会决定你研究的问题的本质

    # 更多类型的图

    Multigraph可以看作是上面提到的加权图，当然我们也可以将两个节点之间的不同的边视为不同的属性。

    # 无向图的连通性

    连通图：任意两个顶点都可以通过一个路径连接。一个非连通图由两个或更多个连通组件组成。

    # 有向图的连通性

    强连接图：图中的任意两点都存在有向路径。 弱连接图：如果不考虑边的方向则是连接的。

    Strongly connected components(SCCs) 可以被识别，但并不是每个节点都是非平凡强连接(去除空集的强连结)组件的一部分。

    • In-component：可以抵达SSC的节点
    • Out-component：可以由SSC抵达的节点
